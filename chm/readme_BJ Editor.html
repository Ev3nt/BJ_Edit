BJ Editor 说明文档


简介：
BJ Editor是一个用于合并war3map.j到Blizzard.j的工具。

地图中脚本只有通过这样合并到Blizzard.j才能再mod中运行，
而这个工具则大大简化了合并过程中需要的操作。
界面说明：


  1. 显示语言（中文/英语）
  2. 按钮
    1. 从地图中提取脚本和一些需要的文件到"Input"文件夹。
    2. 加载"Input"文件夹中的Blizzard.j和输入各个种族的数据，并对加载后的Blizzard.j进行处理，自动生成产生初始单位这类的函数。
    3. 从war3mapMisc.txt中读取自定义英雄，并限制其在游戏中数量。
    4. 读取war3map.j并进行合并。
    5. 读取war3map.wts，替换jass中的字符串标签为真正的字符串。
    6. 保存文件在Output\Blizzard.j中。
    7. 调用pjass进行语法检查。(一般来说语法检查不出错的blizzard.j是能够在mod中运行的)
  3. 函数的注释
  4. 函数主体
  5. 函数列表
  6. 编辑面板，通过上面三个面板进行对载入的Blizzard.j的修改。
  7. 杂项，一些测试性的功能。
如何使用：
1.从1-8依次按下每个按钮。（如果你已经提取出了war3map.j这一步可以忽略）
1-1. 命名为"WEonly_触发器名"的触发将不会被添加进bj中


2.在按下按钮[2]时会弹出如下对话框：

在"种族名称"文本框输入要添加的种族，用逗号隔开（例如：Naga,IceTroll），点击相应文字选取种族，可在下面面板中更改其对应的数据（主基地ID、工人ID、ai文件的名字等）
勾选"Just Load"使程序只加载input文件夹中的Blizzard.j但不对其做任何修改。


3.保存文件，不放心可以按按钮（7）进行语法检测，然后手动修改错误。
4(可选).按下"Pack"按钮打开Pack'n，从而可以将生成脚本导入mod并在游戏中进行测试。
